This code would be impossible to predict reliably because it uses a much truer randomness than what an MLB pitcher/catcher/manager can come up with while also trying to pay attention to everything else happening in the game. The average probability of Itcher selecting the pitch it ends up selecting is 18.2%, which provides a hard ceiling for the accuracy of any human or machine trying to predict it.

Along with the randomness, the pitches this model suggests would perform better than the pitches selected in the dataset. Sticking to the framework of measuring outcomes based on the binary of whether they were good or bad for the pitcher, unsurprisingly, as currently chosen, a given pitch is good for the pitcher about 50% of the time. The specific figure is 51.4%. Based on the machine learning's valuation, the pitch the model selects would give the pitcher a good outcome 53.3% of the time. The optimal pitch would give the pitcher a good outcome 64.5% of the time, but it would be too predictable to use.

The model does have a slight bias, which is evident in the fact that, of the pitches the pitcher actually threw, the model predicted they would have a good outcome 49.6% of the time despite the actual figure being 51.4%. We account for this by adjusting the average predicting on Itcher's selected pitch up by 0.514 - 0.496 = 0.018, meaning the actual average prediction is 0.533 + 0.018 = 0.551.

However, we still have to factor in the model's inaccuracy. The model uses a bagging ensemble, which means the 0.551 roughly translates to 551 models predicting a good outcome while 1000 - 551 = 449 models predict a bad outcome (we don't actually use 1000 models but this is just for illustration). The average child model has an accuracy of 71.1%, and if 71.1% of those 551 "good" models are right, that results in 0.711 * 551 = 391.89 good outcomes, and (1 - 0.711) * 551 = 159.11 bad outcomes. When we do the same with the "bad" models, we get 0.711 * 449 = 319.34 bad outcomes and (1 - 0.711) * 449 = 129.66 good outcomes. This means, adjusting for the inaccuracy, we get about 391.89 + 129.66 = 521.55 good outcomes, i.e. a 52.2% chance of a good outcome. This estimate is more conservative, which makes sense since it is taking the model's output with a grain of salt.

We can convert these figures to a run value by measuring the wOBA before and after the pitch, and then using that to find the wRAA over a full season. Note that the training data is originally categorized into "good" and "bad" events based on this difference in wOBA, although the magnitude is lost. 

For pitches that resulted in a "good" event, the average wOBA difference before and after is -0.1453 (negative is good for the pitcher). Pitches that resulted in a "bad" event averaged a wOBA difference of 0.1552. The selected pitch thus has an average wOBA difference of 0.514 * -0.1453 + (1 - 0.514) * 0.1552 = 0.0008, which is unsurprisingly basically zero. On the other hand, this code's selected pitch has an average wOBA difference of 0.522 * -0.1453 + (1 - 0.522) * 0.1552 = -0.001517263331133431.

In the dataset Itcher uses, there was an average of 3.87 pitches per plate appearance. Each pitch Itcher suggests lowers the batter's wOBA by 0.0015, meaning over the course of a full plate appearance, this accumulates to a difference of 0.0015 * 3.87 = 0.005872.

The league-average pitch by definition allows the league-average wOBA. This project uses 2019 data, and in 2019 the league-wide wOBA was 0.320, the wOBA scale was 1.157, and the number of runs per plate appearance was 0.126 (all figures from [fangraphs](https://www.fangraphs.com/guts.aspx?type=cn)). A plate apperance using pitches selected from Itcher would thus have a wOBA of 0.320 - 0.00587 = 0.314.

In 2019, the average team faced 38.39 batters per game (from [here](https://www.baseball-reference.com/leagues/majors/pitch.shtml)), meaning 38.39 * 162 = 6219.18 per season. To see how valuable a 0.314 wOBA over 6219.18 plate appearances is, it's easiest to convert the figures to wRAA. Using [this formula](https://library.fangraphs.com/offense/wraa/), we find wRAA = ((0.314 - 0.320) / 1.157) * 6219.18 = -31.56252175072624, which is to say that a team using Itcher over a full season would allow something on the order of 31.56 fewer runs than the same team not using Itcher.

Note that this is a somewhat different calculation than what a sabermetrician is typically used to when converted a *player's* wOBA to a run value. Because the team can use Itcher for every pitcher in every game, a smaller difference in wOBA adds up to a much larger difference in runs compared to a single starting pitcher with a 0.314 wOBA allowed.

Fangraphs calculates the amount of runs per win using the formula [here](https://library.fangraphs.com/misc/war/converting-runs-to-wins/). In 2019, this figure was 4.83 * 1.5 + 3 = 10.245 (runs per game from [here](https://www.baseball-reference.com/leagues/majors/pitch.shtml)), meaning that runs saved figure translates to 31.5625218 / 10.245 = 3.0807732 wins over a full season.

Using Itcher provides a team with 3.08 extra wins. If Itcher were a free agent, it would be worth $6.37 million per win * 3.08 wins = $19.62 million dollars in 2022 for just one year. (Dollars per win from [here](https://blogs.fangraphs.com/what-did-teams-pay-per-win-in-free-agency/).)
